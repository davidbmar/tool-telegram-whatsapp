Title: Add HTML views for /status and /projects endpoints
Date: 2026-03-19
Author: agentA

## Goal

Add human-readable HTML views for /status and /projects endpoints when accessed from a browser (Accept: text/html), while keeping JSON as the default for API clients.

## Context

Sprint 5 — B-008 and F-014 requested HTML views. The server previously only returned JSON for these endpoints.

## Plan

1. Add `_wants_html()` helper to check Accept header
2. Update `_handle_status()` to render HTML with green/red health indicators
3. Update `_handle_projects()` to render HTML table with badges for notify events
4. Shared dark theme CSS and nav bar matching config_ui.html
5. Add tests for both JSON and HTML responses

## Changes Made

- `whatsup/server.py`: Added `_wants_html()`, `_NAV_HTML`, `_CSS` class attributes, and HTML rendering branches in `_handle_status()` and `_handle_projects()`
- `tests/test_server.py`: Added `TestStatusEndpoint` (3 tests) and `TestProjectsEndpoint` (4 tests) covering JSON defaults, HTML rendering, nav links, and config link

## Decisions Made

- Used content negotiation (Accept header) rather than separate routes — keeps API surface clean
- Embedded CSS as a class attribute rather than a separate file — keeps it self-contained and consistent with the existing index page pattern
- Reused the same CSS custom properties from config_ui.html for theme consistency
